Bienvenue dans les catégories d'outils de science des données.Après avoir visionné cette vidéo, vous pourrezlister les tâches qu'un data scientist doit effectuermontrer comment la gestion des actifs de code et la gestion des actifs de données aident à créer des modèles, etdécrire comment les environnements d'exécution et de développement implémentent un modèle.Avant que cela puisse être utile,les données brutes doivent passer par différentes catégories de tâches de Data Science, telles quegestion de données,intégration et transformation de données,visualisation de données,modélisme,déploiement du modèle,et la surveillance et l'évaluation des modèles.Pour effectuer ces tâches, vous avez besoin de la gestion des actifs de données, de la gestion des actifs de code,environnements d'exécution et environnements de développement.Voyons comment chaque catégorie vous permet de tirer le meilleur parti des données brutes.La gestion des données est le processus decollecter, conserver et récupérer des données de manière sécurisée, efficace et rentable.Les données sont collectées à partir de nombreuses sources, telles queTwitter, Flipkart, médias, capteurs, et plus encore.Stockez les données collectées dans un stockage persistant afin qu'elles soient disponibles chaque fois que vous en avez besoin.Intégration et transformation de données,est le processus d'extraction, de transformation et de chargement des données.C'est ce qu'on appelle "ETL".Certaines de ces données sont réparties dans plusieurs référentiels.Par exemple, une base de données, un cube de données et des fichiers plats.Utilisez le processus d'extraction pour extraire des données de ces nombreuxréférentiels et enregistrez-les dans un référentiel central tel qu'un entrepôt de données.Les entrepôts de données sont principalement utilisés pour collecter et stocker des quantités massives de données pour l'analyse des données.Ensuite, la transformation des données est le processus de transformation des valeurs, de la structure et du format des données.Après avoir extrait les données, l'étape suivante consiste à transformer les données.Dans cet exemple, les données de taille et de poids doivent être transformées en unités métriques.Et une fois les données transformées,il est temps de charger les données. Les données transformées sont rechargées dans l'entrepôt de données.La visualisation des données est la représentation graphique des données et des informations.Vous pouvez utiliser la visualisation pour représenter des données sous forme de graphiques, de tracés, de cartes, d'animations, etc.Et la visualisation des données transmet les données plus efficacement aux décideurs.C'est une étape cruciale dans le processus de science des données.Diverses formes de visualisations de données incluentun histogramme, qui compare la taille de chaque composant,un treemap, qui affiche les données de la hiérarchie,un graphique en courbes, qui trace une série de points de données au fil du temps,et un graphique cartographique, qui affiche les données par emplacement.Map charts can also be applied to other locations like websites. Now, model building is the next step. This is where you train the data and analyze patterns with machine learning algorithms. The system ‘learns’ how to provide predictions or decisions by itself. You can then use this model to make predictions on new, unseen data. Model building can be done using a service called IBM Watson Machine Learning. It provides a full range of tools and services for building models. The next step is model deployment: the process of integrating a developed model into a production environment. In model deployment, a machine learning model is made available to third-party applications via APIs. Business users can access and interact with the data through these third-party applications. And this helps them make data-based decisions. As an example, the SPSS Collaboration and Deployment Services can be used to deploy any type of asset created by the SPSS software tools suite.
